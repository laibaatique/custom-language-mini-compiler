# FAST Mini Compiler

A Compiler Construction course project implemented in C++. The project covers core compiler phases including lexical analysis, syntax analysis, and intermediate code generation for a custom language used in the course.

## Overview

This repository contains coursework developed across multiple project phases:

- **Phase 1:** Table-driven lexical analyzer (scanner)
- **Phase 2:** Recursive-descent parser with parse tree generation
- **Phase 3:** Intermediate code generation using a translation scheme to produce three-address code

The project is based on a custom language specification provided in the course, with reserved words such as `Adadi`, `Ashriya`, `Harf`, `Matn`, `Mantiqi`, `Agar`, `Wagarna`, `Marqazi`, `input->`, and `output<-`.

## Project Features

### Lexical Analyzer
- Table-driven finite automaton implementation
- Custom transition handling using user-defined data structures
- Tokenization for:
  - identifiers
  - keywords
  - numbers
  - operators
  - punctuations
  - string literals
- Generates:
  - token stream
  - symbol table
  - literal table
  - lexical error file
- Handles comments and string literals
- Uses double-buffer style input processing in the lexer implementation

### Parser
- Recursive-descent parser in C++
- Reads token stream generated by the lexer
- Validates syntax according to the project grammar
- Prints parse tree / production expansion output
- Supports constructs such as:
  - function structure
  - declarations
  - expressions
  - compound statements
  - `for`
  - `while`
  - `if-else`

### Intermediate Code Generation
- Generates three-address code for a restricted grammar
- Handles:
  - arithmetic expressions
  - assignment expressions
  - relational expressions
  - `if-else`
  - `while`
  - compound statements
- Produces TAC output in a `.tac` file

## Language Specification Summary

According to the Phase 1 project definition, the scanner processes categories such as identifiers, signed/unsigned/floating/exponential numbers, punctuations, operators, and keywords defined for the custom compiler language.

Examples from the language include:
- `Adadi`
- `Ashriya`
- `Harf`
- `Matn`
- `Mantiqi`
- `Agar`
- `Wagarna`
- `Wapas`
- `Marqazi`
- `input->`
- `output<-`
